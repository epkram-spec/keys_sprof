# AGENTS.md

## Проєкт
Ти розробляєш внутрішній вебсервіс “Радар кейсів SPROF”.
Технічна назва може бути SPROF Case Radar, але в інтерфейсі показувати тільки “Радар кейсів SPROF”.

## Мета
Сервіс потрібен тільки для збору, оцінки, ведення і передачі потенційних кейсів у маркетинг.
Це НЕ CRM, НЕ система продажів, НЕ таск-менеджер, НЕ склад, НЕ контент-план.

## Перший адміністратор
Перший користувач-адміністратор: epkram@gmail.com.

## Стек
Next.js App Router, TypeScript, Tailwind CSS, shadcn/ui, Supabase Auth, Supabase PostgreSQL,
Supabase RLS, Supabase Storage, Vercel, Google Sheets API, email notifications, Telegram Bot notifications.

## Мова
Увесь видимий інтерфейс українською.
Не показувати користувачу: Login, Dashboard, Cases, Create, Save, Cancel, Delete, Error, Required,
Invalid, Success, Admin, Settings, Upload, Export, Import, Status, Priority.
Технічні назви в коді можуть бути англійською, але користувач їх не бачить.

## Ролі
Дозволені ролі: Менеджер, Маркетинг, Керівник, Адміністратор.
Не додавай нові ролі без прямої інструкції.

## Сповіщення
Усі сповіщення мають відображатись у кабінеті.
У хедері має бути дзвіночок з кількістю нових, не переглянутих сповіщень.
Нові сповіщення мають статус unread / не переглянуте.
Старі сповіщення мають статус read / переглянуте.
Email і Telegram є додатковими каналами, але in-app запис створюється завжди.
Якщо Telegram не привʼязаний, не блокувати сповіщення: лишити in-app і email.
Не дублювати однакові нагадування. Використовувати dedupe_key.

## GitHub / DB / Vercel
Працюй по Gate-етапах у окремих гілках.
Не пуш у GitHub без команди власника: “Погоджую Gate X. Коміть і пуш”.
Усі зміни БД роби тільки через supabase/migrations.
Не застосовуй міграції на staging або production без окремого погодження.
Не запускай Vercel staging/preview без окремого погодження.
Production deploy тільки після окремого підтвердження.

## Заборони
Не додавай CRM, продажі, склад, контент-план, AI-генерацію текстів, складний календар зйомок,
задачі для команди.
Не змінюй ролі, статуси, поля і логіку без прямої інструкції.
Не видаляй файли без явного підтвердження.
Не коміть секрети.
